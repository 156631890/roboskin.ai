# RoboSkin Brand Homepage Redesign Design

## Design Read

RoboSkin.ai is an authority portal for researchers, robotics builders, and technical readers tracking robot skin, tactile AI, e-skin, tactile sensors, and physical AI touch. The redesign should read as a premium dark technical brand system, not a generic AI SaaS landing page.

## Scope

Preserve the existing route structure, primary navigation labels, SEO copy, JSON-LD behavior, and conservative claim discipline. Redesign the homepage, shared visual tokens, navigation treatment, and homepage visual components only. Existing research, glossary, resources, and contact paths remain the conversion structure.

## Visual System

Use a deep graphite and cold-silver base with one locked electric blue/cyan accent family. The page should feel like a tactile robotics lab, a signal-analysis instrument, and a research index. Avoid warm luxury palettes, purple-blue AI gradients, generic three-card feature rows, invented product claims, and unsupported metrics.

The first viewport should have a stronger brand signal: a large image-led robotic tactile-skin scene, disciplined headline hierarchy, clear research/glossary CTAs, and a visible preview of the downstream stack/index content. Downstream sections should vary rhythm: editorial index, direct answers, research briefs, visual public assets, market signals, and final contact route.

## Assets

Generate new ImageGen assets for the homepage brand layer:

- A brand concept board used as visual reference.
- A wide hero image for the homepage opening scene.
- A tactile stack detail image for the homepage stack map if the existing image does not carry enough brand presence.

Final project-bound assets live under `public/generated/brand/` and are referenced from `src/content/site.ts`.

## Verification

Run existing source tests, lint, build, and browser inspection at desktop and mobile widths. The redesign must keep the tested homepage copy strings and asset wiring intact while adding the new brand asset contract.
